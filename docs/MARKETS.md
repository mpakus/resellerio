# Marketplace Notes

Checked: 2026-03-30

Purpose:

- working reference for future marketplace title, description, and tag generation
- policy-aware guidance only; not legal advice
- always re-check live seller UIs and official policies before auto-publishing

General generation rules across every channel:

- keep copy factual, product-specific, and brand-safe
- never invent condition, measurements, authenticity, sustainability, provenance, or warranty claims
- never use competitor brand names, irrelevant keywords, or copied third-party text for search gain
- never include off-platform payment/contact instructions unless the platform explicitly requires it
- treat `generated_tags` as structured keywords first; only render literal `#hashtags` where the marketplace clearly supports that pattern

Legend:

- `Seller listing`: seller writes and owns the public listing copy
- `Peer-to-peer`: seller writes the public listing, but the platform is strongly local or messaging-driven
- `Managed consignment`: seller submits inventory, but the platform heavily controls merchandising, copy, or acceptance

## Implementation Notes

- `generated_tags` should stay platform-neutral in storage. Render them as literal hashtags only for channels where the official product and policy guidance clearly support that presentation.
- `facebook_marketplace`, `offerup`, `whatnot`, and `grailed` should default to concise plain-text descriptions with no inline hashtag decoration.
- `etsy` should map `generated_tags` into Etsy tag fields later, not into the listing title or description.
- `therealreal` and `thredup` are best treated as intake-routing targets rather than pure self-serve marketplace-copy targets.
- Re-check live product forms before shipping any hard-coded title or description length limits. Public docs were inconsistent or incomplete for several marketplaces.
