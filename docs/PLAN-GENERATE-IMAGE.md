## Current Status

GI3–GI6 are shipped. Seller-triggered generation, scene regeneration, approval, and deletion work from both the product LiveView and the JSON API (`/api/v1/products/:id/generate_lifestyle_images`, etc.).

Auto-generation during the AI pipeline is gated behind `Reseller.AI.lifestyle_generation_enabled?/1` and disabled by default.

Key modules:
- `Reseller.AI.ScenePlanner` — maps product category to a `scene_family`
- `Reseller.AI.LifestylePromptBuilder` — builds per-scene prompts with negative rules
- `Reseller.AI.GeneratedImage` — normalizes Gemini image responses for storage
- `Reseller.Workers.LifestyleImageGenerator` — orchestrates 3 separate Gemini requests per generation run

## Open: Step GI7 — Cost Controls, Rate Limits, Observability

- Premium model fallback config (`gemini-3.1-flash-image-preview`) for seller-triggered re-runs
- A/B prompt version tracking

## Generation Notes

- Source image priority: `background_removed` → original fallback → legacy `white_background`
- Each generation run fires 3 separate Gemini requests (one per scene brief) for diversity and isolated failure handling
- Generated bytes stored as `product_images` with `kind: "lifestyle_generated"`
- Default output: `4:5` aspect ratio, `1K` size
- All Gemini-generated images include SynthID watermarking
